Server Configuration Guide

⚠️ Web Preview Not Working

VS Code's Simple Browser preview has limitations. To run the application properly, you need to set up a local HTTP server using one of the methods below.

🚀 Setup Instructions

Option 1: VS Code Live Server (Recommended)

Step 1: Install the "Live Server" extension

  • Open VS Code Extensions (Ctrl+Shift+X)
  • Search for "Live Server" by Ritwick Dey
  • Click Install

Step 2: Start the server

  • Right-click on index.html
  • Select "Open with Live Server"
  • Browser opens automatically at http://127.0.0.1:5500

✨ Best option: auto-refreshes on file changes

Option 2: Python HTTP Server

Requirements: Python 3.x installed

Step 1: Open terminal in project directory

Step 2: Run this command:

python -m http.server 3000

Step 3: Open browser to http://localhost:3000

Option 3: Node.js Server

Requirements: Node.js installed

Step 1: Open terminal in project directory

Step 2: Run this command:

node server.js

Step 3: Open browser to http://localhost:3000

Option 4: Windows Batch File

Step 1: Double-click start-server.bat

Step 2: A terminal window opens

Step 3: Server automatically starts on http://localhost:3000

✨ Simplest for Windows users

Option 5: PowerShell HTTP Server

Step 1: Open PowerShell in project directory

Step 2: Run this command:

Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process; .\http-server.ps1

Step 3: Server starts on http://localhost:8000

❓ FAQ

Q: Why doesn't VS Code preview work?

A: VS Code's Simple Browser has limitations with localStorage and external API requests needed for this app.

Q: Which method should I use?

A: Live Server extension is easiest. It auto-refreshes and is ideal for development.

Q: Can I deploy this?

A: Yes! The app is ready for Vercel, Netlify, or any static hosting. The /api/hls.js works on Vercel.

Q: Do I need internet?

A: Yes, the app needs internet to fetch channel lists from iptv-org API and stream content.
